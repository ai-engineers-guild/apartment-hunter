# Apartment Hunter

Apartment Hunter это библиотека и MCP-сервис для поиска арендного жилья.
Общее ядро сделано source-agnostic, а специфика площадок вынесена в адаптеры,
например в `krisha.kz`.

## Возможности

- сбор объявлений из адаптеров источников;
- нормализация в общую модель `Apartment`;
- хранение квартир, профилей, уведомлений и истории цен;
- семантический поиск и LLM-анализ;
- MCP-интерфейс для Codex Desktop и других MCP-клиентов.

## Структура

`src/apartment_hunter/core`
- общие модели и интерфейсы.

`src/apartment_hunter/adapters`
- адаптеры и парсеры конкретных площадок.

`src/apartment_hunter/ingest`
- оркестрация загрузки, дедупликации, анализа, индексации и уведомлений.

`src/apartment_hunter/storage`
- хранилища и vector stores.

`src/apartment_hunter/mcp`
- MCP tools для поиска, ингеста, мониторинга и анализа.

`src/krisha`
- legacy-код старого проекта, отделенный от переиспользуемой библиотеки.

## Основные команды

```powershell
uv run pytest
uv run pytest --cov
uv run ruff check .
uv run apartment-hunter-mcp
uv run apartment-hunter-ingest
```

## Интеграция с Codex Desktop

Обычный сценарий:

1. запустить MCP сервер: `uv run apartment-hunter-mcp`;
2. создать профиль поиска;
3. запустить ингест;
4. спрашивать у агента:
   что нового,
   что изменилось в цене,
   какие квартиры лучшие,
   что подходит под словесное описание,
   чем отличаются варианты.

Локальные skills лежат в `.agents/skills`:

- `apartment_search`
  Доменный workflow по поиску квартир и переводу жаргона объявлений на язык агента.
- `city-district-context-kz`
  Контекст по городам и районам Казахстана: транспорт, цены, благополучие,
  стройки, шум, удобство жизни.

## Архитектурные правила

- общее поведение живет в core;
- site-specific логика живет в adapters;
- MCP должен использовать библиотеку, а не подменять ее;
- `is_new` означает первое обнаружение квартиры;
- delta последнего запуска берется из результата pipeline, а не из грубой выборки по базе.
